# Context7-Style Documentation Platform: Top 100 Libraries Research

**Research Date:** November 2025
**Purpose:** Identify and prioritize the most popular software libraries/frameworks for documentation indexing

---

## Executive Summary

This research identifies the top 100 most popular software libraries and frameworks across multiple programming languages for a Context7-style documentation indexing platform. The analysis is based on:

- **Download metrics**: npm weekly downloads, PyPI monthly downloads, Maven downloads
- **GitHub stars**: Community engagement and popularity indicators
- **Developer surveys**: Stack Overflow Developer Survey 2024-2025, State of JavaScript 2024
- **Documentation quality**: Structure, versioning, and maintainability

### Key Findings

- **Total estimated pages**: ~15,000-25,000 documentation pages across top 100 libraries
- **Primary languages**: JavaScript/TypeScript (40%), Python (25%), Java (15%), Others (20%)
- **Documentation formats**: Static sites (Docusaurus, Nextra, MkDocs), API docs (Swagger, JSDoc), ReadTheDocs
- **Update frequency**: Varies from daily (major frameworks) to quarterly (stable libraries)

## Crawl Requirements Analysis

### Total Pages Estimation

```
Tier 1 (Score 95-100): 20 libraries × 600 pages avg = 12,000 pages
Tier 2 (Score 85-94): 30 libraries × 500 pages avg = 15,000 pages
Tier 3 (Score 75-84): 35 libraries × 300 pages avg = 10,500 pages
Tier 4 (Score 65-74): 15 libraries × 200 pages avg = 3,000 pages

TOTAL ESTIMATED PAGES: 40,500 pages
```

### Storage Requirements

Assuming average documentation page:
- **Raw HTML**: 50 KB per page
- **Processed Markdown**: 20 KB per page
- **Metadata**: 5 KB per page

```
Total Raw Storage: 40,500 pages × 50 KB = 2,025 MB (~2 GB)
Total Processed Storage: 40,500 pages × 20 KB = 810 MB
Total Metadata: 40,500 pages × 5 KB = 202.5 MB
```

### Embedding Generation Costs

Using OpenAI's text-embedding-3-small model:

**Assumptions:**
- Average page: 500 tokens
- Total tokens: 40,500 pages × 500 = 20,250,000 tokens

**Pricing (text-embedding-3-small):**
- Standard tier: $0.02 per 1M tokens
- Batch tier: $0.01 per 1M tokens

```
Standard Tier Cost: 20.25M tokens × $0.02 = $0.405 (~$0.41)
Batch Tier Cost: 20.25M tokens × $0.01 = $0.2025 (~$0.20)
```

**Re-indexing frequency:**
- Monthly updates: $0.41/month (standard) or $0.20/month (batch)
- Quarterly updates: $0.41/quarter
- Selective updates (20% monthly): $0.08/month

### Vector Database Storage

**Vector dimensions:** 1536 (text-embedding-3-small)
**Storage per vector:** ~6 KB (vector + metadata)

```
Total Vector Storage: 40,500 vectors × 6 KB = 243 MB
```

**Popular Vector Database Options:**
1. **Pinecone**: ~$70/month (100K vectors, can hold all)
2. **Weaviate**: Self-hosted or cloud (~$50/month)
3. **Qdrant**: Self-hosted (free) or cloud (~$45/month)
4. **Chroma**: Self-hosted (free)

### Initial Crawl Timeline

**Assumptions:**
- Crawl rate: 1 page per second (respectful crawling)
- Processing time: 0.5 seconds per page
- Total time per page: 1.5 seconds

```
Total Crawl Time: 40,500 pages × 1.5 sec = 60,750 seconds
= 1,012.5 minutes
= ~17 hours of continuous crawling
```

**Recommended approach:**
- Parallel crawling (5-10 concurrent workers)
- Estimated time: 2-4 hours for initial index

### Ongoing Maintenance

**Weekly monitoring:**
- Check 20% of libraries for updates (~8,100 pages)
- Time: ~3 hours/week
- Embedding cost: ~$0.08/week

**Monthly full refresh:**
- Re-crawl all libraries
- Time: ~4 hours
- Embedding cost: $0.20-$0.41

## Technical Recommendations

### Crawling Strategy

1. **Respect robots.txt and rate limits**
2. **Use sitemap.xml when available**
3. **Implement exponential backoff**
4. **Cache aggressively**
5. **Monitor for documentation updates via:**
   - GitHub release webhooks
   - RSS feeds
   - Version API endpoints

### Content Processing

1. **HTML to Markdown conversion**
2. **Code block extraction and syntax highlighting**
3. **Link normalization**
4. **Image optimization**
5. **Metadata extraction:**
   - Version
   - Last updated
   - Category/tags
   - Framework version compatibility

### Search Optimization

1. **Multi-modal embeddings:**
   - Text content
   - Code snippets
   - Headings (weighted higher)

2. **Hybrid search:**
   - Vector similarity (semantic)
   - BM25 (keyword)
   - Popularity boosting

3. **Faceted filtering:**
   - By library
   - By version
   - By category
   - By language
